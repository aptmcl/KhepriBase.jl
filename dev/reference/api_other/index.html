Backend Developer API

KhepriBase.@import_backend_api is the entry point for backend packages: it splices every public-but-not-exported KhepriBase name into the calling module so backends can extend b_sphere, b_trig, etc., and otherwise see the full developer surface without having to maintain an import list. See Implementing a Backend for usage context.

The multi-storey Layout, its Storeys and Spaces, and every accessor used by the constraint library and the declarative engine live here. Narrative and examples are in the Spaces BIM chapter; the imperative and declarative compilation paths are covered in the Layout Engine reference.

Output of build(storey): the emitted BIM elements plus the IFC-style SpaceBoundary map. Supports tuple destructuring: walls, doors, windows, slabs = build(storey).

KhepriBase.add_door — Method
add_door(storey_or_layout, space_a, space_b; family, loc, sill=0)

Attach a SpaceConnection of kind :door between space_a and space_b (which may be another Space or the symbol :exterior). The Layout form routes to whichever storey owns space_a. loc is required for exterior doors; interior doors default to mid-wall placement at build time. Doors sit at floor level (sill = 0); pass a non-zero sill for raised thresholds.

source
KhepriBase.add_rule — Method

Attach a validation Constraint to a Layout (constraints live on the Layout so they can span storeys).

source
KhepriBase.add_space — Method
add_space(storey_or_layout, name, boundary; kind=:space)

Create a Space with the given name, boundary, and kind, and add it to the target. Passing a Storey targets that storey directly; passing a Layout routes to its last storey (the common single-storey case) and invalidates the space-lookup cache.

source
KhepriBase.add_storey! — Method

Append a Storey to a Layout. Defaults the new storey's level to one above the topmost existing storey.

source
KhepriBase.add_window — Method
add_window(storey_or_layout, space_a, space_b; family, loc, sill)

Analogue of add_door for windows. sill is the bottom-edge elevation above the floor (metres); defaults to default_window_sill_height (0.9 m residential sill). Pass sill=0 for floor-to-ceiling glazing.

source

source

KhepriBase.build — Method
build(storey_or_layout)

Compile a Storey (or every storey of a Layout) down to BIM elements via the wall-graph chain resolver: shared edges become walls, interior connections become doors/windows/arches, boundaries become SpaceBoundary records. Returns a BuildResult (single storey) or Vector{BuildResult} (layout).

source

KhepriBase.collinear_overlap — Function

Parametric overlap of two collinear directed edges (a1→a2) and (b1→b2). Returns (t_start, t_end) on the a edge, or nothing if non-collinear or non-overlapping.

source
KhepriBase.exterior_edges — Function

Exterior-facing edges of a space within a storey as (Loc, Loc) pairs. Used for facade computations.

source
KhepriBase.find_space — Method

source
KhepriBase.floor_plan — Method

KhepriBase.layout — Method
