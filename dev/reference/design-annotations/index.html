Annotations

Annotations attach metadata to a SpaceDesc subtree without changing its geometry. Downstream consumers (such as an element generator or the build(layout) pipeline) inspect annotations to decide where to place doors, which walls stay solid, and which spaces opt out of automatic windows.

house = (room(:living, :living_room, 5.0, 4.0) |
         room(:kitchen, :kitchen, 3.5, 4.0)) |>
  d -> connect(d, :living, :kitchen; kind=:arch, width=1.8) |>
  d -> connect_exterior(d, :living; kind=:door, face=:south) |>
  d -> no_windows(d, :kitchen)                 # kitchen is an interior galley

# Remove a default connection between neighbouring rooms:
disconnect(house, :living, :kitchen)

Nested Annotated wrappers compose transparently: layout walks through them and collect_annotations gathers them for downstream use.

Types

KhepriBase.Annotation — Type
Annotation

Abstract supertype for annotations attached to a space description tree. Annotations describe connections, disconnections, and other metadata.

source
KhepriBase.ConnectAnnotation — Type
ConnectAnnotation <: Annotation

Declares a connection (door, window, or arch) between two named spaces.

source
KhepriBase.ConnectExteriorAnnotation — Type
ConnectExteriorAnnotation <: Annotation

Declares an exterior opening (door or window) on a specific face of a space.

source
KhepriBase.DisconnectAnnotation — Type
DisconnectAnnotation <: Annotation

Removes any default connection between two adjacent spaces.

source
KhepriBase.NoWindowsAnnotation — Type
NoWindowsAnnotation <: Annotation

Suppresses automatic window generation for the given space.
