Tree Queries

Pure functions for inspecting a SpaceDesc tree before laying it out. They return the bounding dimensions of the composed tree and the set of ids and annotations it carries — useful when writing constraints, custom combinators, or debugging a design.

house = (room(:living, :living_room, 5.0, 4.0) |
         room(:kitchen, :kitchen, 3.0, 4.0)) /
        (room(:bed, :bedroom, 4.0, 3.0) |
         room(:bath, :bathroom, 2.5, 3.0))

desc_width(house)   # 8.0
desc_depth(house)   # 7.0
desc_height(house)  # 2.8 (default room height)
collect_ids(house)  # [:living, :kitchen, :bed, :bath]

KhepriBase.update_room_by_id — Function
update_room_by_id(desc, id, f)

Walk a SpaceDesc tree and return a new tree with the Room or Carved node whose id matches id rewritten by f(node) -> node. All other nodes are preserved by structure. Cells inside GridLayout and zone targets inside Refined are opaque (their function argument isn't inspected) and are passed through unchanged.
